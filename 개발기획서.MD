# 개발 기획서: 부울경 소상공인 AI 마케팅 매니저

## 1. 목표와 핵심 가치
- 목표: 사진/간단 정보만으로 인스타/스토리/지도 리뷰/문자 홍보문구를 자동 생성하는 로컬 특화 도구
- 핵심 가치: 글 잘 못 써도 매일 홍보 가능, 부산 데이터·감성 반영, 빠른 사용성
- 주요 타깃: 부산·울산·경남 소상공인(카페, 식당, 동네 가게 등)

## 2. 제품 범위
- 필수(MVP)
  - Vite+React 단일 페이지, 입력 폼 및 결과 표시
  - POST /api/generate 연동: 채널별 문구/해시태그 생성, 복사 기능
- 선택(시간 되면)
  - Tone & Manner 분기(서울말/부산 사투리), “오늘 우리 동네 이슈” 버튼 + 로컬 컨텍스트 주입
  - 탭 UI 다듬기, 간단 통계/로그 저장
- 여유 시
  - 사진 업로드 → 사진 설명 필드 반영, 마케팅 캘린더 UI 확장, 생성 히스토리 화면

## 3. 기술 스택
- Frontend: React + Vite, TypeScript, Tailwind CSS, React Query(+Context), fetch/axios
- Backend: Node.js + Express(BFF), LLM API 프록시, 로컬 데이터 적재(JSON/CSV), 간단 로깅
- AI/데이터: 업스테이지 Solar API, 부산 축제/유동인구/관광 데이터, 날씨/요일/시간대 컨텍스트

## 4. 아키텍처 개요
- 브라우저(SPA) ↔ REST API ↔ BFF(Express)
  - BFF: LLM 호출, 부산 데이터 로딩/필터링, 프롬프트 생성·포맷팅, 응답 캐싱(옵션)
- 데이터: 사전 로딩된 로컬 JSON/CSV → location/date 기준 필터 → 프롬프트 컨텍스트 주입

## 5. 주요 기능
- AI 홍보글 생성기: 업종/가게명/위치/소개/대표메뉴/이벤트/Tone/시점/채널 입력 → 채널별 문구+해시태그 반환
- 로컬 이슈 추천: “오늘 우리 동네 이슈 불러오기” → location+date 필터 후 컨텍스트 제공
- 마케팅 시점 선택: 오늘/주말/급여일/축제 버튼값을 프롬프트에 반영
- 결과 표시: 채널별 탭/아코디언, 복사 버튼, 로딩/에러 표시

## 6. 화면 설계(요약)
- 메인: 헤더(타이틀+슬로건) / 입력 카드 / “이슈 불러오기” 버튼 / “AI에게 맡기기” 버튼 / 결과 탭(인스타 피드·스토리·지도 리뷰·문자)
- 결과: 각 채널 블록에 생성 문구, 해시태그(인스타), 복사 버튼
- 푸터: 트랙/데모 안내 간단 표기

## 7. API 설계(초안)
- POST /api/generate
  - Req: storeName, category, location, intro, menus[], event, tone, timing, channels[], useLocalContext
  - Res: contextSummary, results{instagram_feed{text,hashtags[]}, instagram_story{text}, map_review{text}, sms{text}}
- GET /api/local-context?location=남포동 (선택)
  - Res: contextSummary("오늘 광안리 불꽃축제…")

## 8. 데이터/프롬프트 전략
- 데이터 전처리: 부산 축제/유동인구/관광 JSON/CSV 사전 로딩, 날짜+동/상권 필터 후 summary 생성
- 프롬프트 템플릿: 가게 정보 + 로컬 컨텍스트 + 시점 + Tone 설명 → 채널별 문구/해시태그를 JSON으로 반환

## 9. 일정/우선순위 가이드(해커톤 9~16시 가정)
- 1단계(~11시): Vite+React 셋업, 메인 레이아웃, 입력 폼 작성, /api/generate 목업 연동
- 2단계(11~13시): 채널별 결과/복사, Tone 분기, 로딩/에러 처리
- 3단계(13~15시): 로컬 컨텍스트 버튼+목업 데이터, UI 다듬기
- 4단계(~16시): 폴리싱, 데모 스크립트, 기본 로그

## 10. 비기능 요구
- 빠른 응답(LLM 호출 최소화, 캐싱 고려), 오류 시 친절 메시지, 데모용 기본 입력값 제공
- 배포: Vite 빌드 → 정적 호스팅, BFF는 단일 Express 서버로 LLM 프록시
